## 设计模式

GOF/Gang of Four/四位作者

设计模式主要是基于以下的面向对象设计原则。

* 对接口编程而不是对实现编程。
* 优先使用对象组合而不是继承。

## 分类

* 创建型模式```Creational Patterns```
    * 工厂方法模式```Factory Pattern```
    * 抽象工厂模式
    * [单例模式```Singleton Pattern```](https://www.cnblogs.com/thomerson/p/16795927.html)
    * 建造者模式
    * 原型模式

* 结构型模式```Structural Patterns```
    * 适配器模式
    * 装饰器模式
    * 代理模式
    * 外观模式
    * 桥接模式
    * 组合模式
    * 享元模式
    
* 行为型模式
    * 策略模式
    * 模板方法模式
    * 观察者模式
    * 迭代子模式
    * 责任链模式
    * 命令模式
    * 备忘录模式

    * 访问者模式
