* 信息系统开发方法⭐⭐⭐
    * 结构化方法/生命周期法：适用于瀑布型
        * **自顶向下，逐步求精，模块化设计**
        * 技术
            * 数据流图DFD/data flow diagram
            * 实体关系图ERD/Entity relation diagram
            * 数据字典DD/data dictionary
            * 系统结构图SSD/system struct diagram
        * 局限性
            * **开发周期长**
            * 难以适应**需求变化**
            * 很少考虑数据结构
        * 适用于数据处理领域
        * 不适用于规模大，较复杂的系统
    * 面向对象开发方法OO/Object-Oriented
        * [面向对象](#面向对象)
        * 适用于**敏捷**
        * 技术
            * [统一建模语言United Model Language/UML](#统一建模语言uml)
        * 优点
            * 更好的**复用**
            * 缩短开发周期
            * 普遍适用于各类系统
        * 不足
            * 需要一定的OO技术支持
            * 大型项目的开发上具有一定的局限性
    * 原型化方法/快速原型法/原型法、Prototyping method
        * 步骤
            * 原型**创建**
            * 用户**体验**
            * 反馈**收集**
            * 原型**修改**
        * 分类
			* 水平/行为原型
                * 探索行为细化需求，未真实实现
                * 主要用户界面上，需求
			* 垂直/结构化原型
                * 实现部门功能
                * 主要用于**复杂算法**，设计
			* 抛弃/探索式原型
                * 尝试犯错
                * 用于解决需求不确定性，二义性，不完整性，含糊性
			* 演化式原型
                * 用于易于升级和优化的场合，特别适合Web项目
        * 优点
            * 开发周期短
            * 增加**用户满意度**
            * 适用于**需求不明确**，分析层面难度大的系统
        * 不足
            * 开发环境要求高
            * 管理水平要求高
            * 不适用于技术层面难度大的系统
    * 面向服务的方法Service Oriented/SO
        * 以服务为核心元素封装企业业务流程
        * 快速相应需求
        * 提高系统可复用性
        * 互操作性
        * 从低到高抽象级别⭐
            * 操作
            * 服务
            * 业务流程

## 信息系统集成技术
###  OSI七层网络模型⭐⭐⭐⭐⭐
开放系统互连参考模型OSI/Open system Interconnect
* 物数网传会表应
* 物理层：物理连接（集线器HUB,调制解调器，中继器）⭐⭐⭐
    * RS232显卡串口
    * RJ-45水晶头
    * V.35调制解调器（MODEM），俗称猫
    * FDDI光缆
* 数据链路层：逻辑连接，物理寻址（交换机，网桥）
    * IEEE802.X⭐⭐⭐
        * 1概而论：通用网络概念及网桥
        * 2萝莉控LLC：数据链路控制
        * 3姨太（以太）：以太网
        * 4线：总线结构及访问方法，物理层规定
        * 5环：Token Ring访问方法及物理层规定
        * 6Man：城域网的访问方法及物理层规定
        * 7宽FDDI：宽带局域网
        * 8光： 光纤局域网
        * 9音：ISDN综合业务数字网局域网
        * 10全：网络安全
        * 11Wlan：Wi-Fi
        * 15.1：蓝牙
* 网络层：路由选择，逻辑寻址（路由器）
    * ARP：地址解析协议，逻辑地址→物理地址⭐⭐⭐
    * RARP：反向地址转换协议，物理地址->逻辑地址⭐⭐⭐
    * ICMP：Internet控制报文协议，控制消息是指网络通不通、主机是否可达、路由是否可用等网络本身的消息⭐⭐⭐
    * IGMP：互联网组管理协议,多播，群发
    * IP协议
* 传输层：端对端的可靠数据传输，打包数据（网关）
    * TCP:传输控制协议 Transmission Control Protocol 
    * UDP:用户数据报协议 User Datagram Protocol
* 会话层：建立连接，管理，终止会话
    * RPC:远程过程调用 Remote Procedure Call
    * SQL:结构化查询语言Structured Query Language
    * NFS:网络文件系统 Network File System
* 表示层：数据加解密，转换，格式化，压缩
    * JPEG
    * ASCII
    * DES
    * MPEG
* 应用层
    * TCP之上
        * FTP:文件传输协议 File Transfer Protocol
        * HTTP:超文本传输协议 Hyper Text Transfer Protocol
        * SMTP⭐⭐⭐Simple Mail Transfer Protocol   简单邮件传输协议
        * Telnet:Internet远程登录服务的标准协议
    * UDP之上
        * TFTP: 简单文件传输协议 Trivial File Transfer Protocol
        * DHCP:动态主机配置协议 Dynamic Host Configuration Protocol⭐⭐
        * DNS: 域名解析 Domain name resolution⭐⭐
        * SNMP: 简单网络管理协议 Simple Network Management Protocol⭐⭐⭐

## 新一代信息技术

### 物联网
the internet of things
解决：物品与物品（T2T），人与物品(H2T)，人与人(H2H)之间的互联
通过信息传感设备（如射频识别RFID），按约定的协议，将任何物品与互联网相连接

* 两项关键技术
    * 传感器技术
    * 嵌入式技术
* 三层架构
    * 感知层
    * 网络层
    * 应用层

### 智慧城市
物联网在城市管理中综合应用
* 五个功能层：联通计数应
    * 物联感知层（通过传感器、监控摄像机、 GPS终端，进行信息采集、识别和监测）
    * 通信网络层（以互联网、电信网、广播电视网为基础，组成网络通信基础设施）
    * 计算与存储层（包括软件资源、计算资源和存储资源，提供数据存储和计算）
    * 数据与服务支撑层（利用SOA、云计算、大数据等技术，提供服务和共享资源）
    * 智慧应用层（智慧交通、智慧旅游等，提供整体的信息化应用和服务） ；
* 三个支撑体系
    * 安全保障体系
    * 标准规范体系
    * 建设和运营管理体系

### 云计算
资源变共享服务、按使用量付费、共享、虚拟化、分布式、易扩展
* IaaS（基础设施即服务）：硬件、存储等变服务；
* PaaS（平台即服务）：操作系统、数据库、 WEB应用等变服务；
* SaaS（软件即服务）：应用软件、组件、工作流等变服务；


### 大数据
* 5V 数据之和的价值远远大于各数据价值之和 
    * volume大量
    * variety多样
    * value价值
    * velocity高速
    * veracity真实性

* 四个环节
    * 数据采集（主要技术数据抽取工具 ETL ）
    * 数据存储
    * 数据管理
    * 数据分析与挖掘

### 区块链
是一种去中心化的分布式账本数据库，是比特币的底层技术
* 6大特征：中开信任自匿信息
    * **去中心化**
    * 开放性
    * 去信任
    * 自治性
    * 匿名性
    * 信息不可篡改
* 分层6层：数网共识，激合应用
    * 数据层：封装了底层数据区块以及相关的数据加密和时间戳等基础数据和基本算法；
    * 网络层：包括分布式组网机制、数据传播机制和数据验证机制等；
    * 共识层：主要封装网络节点的各类**共识算法**；
    * 激励层：将经济因素集成到区块链技术体系中来，主要包括经济激励的发行机制和分配机制等；
    * 合约层：主要封装各类脚本、 算法和智能合约，是区块链可编程特性的基础；
    * 应用层：封装了区块链的各种应用场景和案例

## 项目论证

### 项目论证 ⭐
* 技术上先进性，适用性
* 经济上合理性，盈利性
* 事实上可能性，风险性
* 全面科学综合分析，为项目决策提供客观依据
* 先论证，后决断
* 市场是前提，技术是手段，财务经济是核心
* 阶段
    * 机会研究：寻求投资机会，鉴别投资方向  ±30%
    * 初步可行性研究：初步评估项目是否有生命力，能否盈利 ±20%
    * 详细可行性研究：详细技术经济论证，在多方案比较的基础上选出最优方案 ±10%
* 步骤（名收 你必选 报告和计划）
    * 明确目标和范围
    * 收集资料
    * 拟定实施方案
    * 分析比较方案
    * 选择最佳论证方案
    * 编制报告
    * 编制计划

### 机会研究
* 寻求有价值的投资机会
* 初步调查和研究
* 项目建议书前期的咨询文件
* 可省略或与初研合二为一
* 输出：项目建议书⭐⭐

### 项目建议书⭐
* 必要性
* 必需条件
* 项目市场预测
* 方案或服务市场预测

### 可行性研究 ⭐⭐⭐⭐⭐
* 技术可行性
    * 技术能力
    * 开发风险
    * 人力资源有效性
    * 物资可用性
* 经济可行性分析
    * 支出分析
        * 一次性和非一次性
    * 收益分析
        * 直接和间接
    * 收益投资比和投资回收期分析
    * 敏感性分析
* 运行环境可行性分析
* 其他方面
    * 法律
    * 社会
    * 财务
    * 组织
    * 投资必要性
    * 风险因素及对策--市场风险，法律等...

* 初步可行性研究报告
* 辅助（功能）研究

        * 财务费用与折扣

    * 投资估算法

    * 市场预测法
* 内容 ⭐⭐⭐⭐⭐
    * 概述：背景，意义，重要性
    * 需求

考虑估算中不确定性和风险

两种假定分布
* 三角分布
期望值=(最悲观+最乐观+最可能)/3
* 贝塔分布
期望值=(最悲观+最乐观+最可能*4)/6
* 标准差/标准偏差σ
反映平均值分散程度，表示精确度的重要指标
σ=(最悲观-最乐观)/6
方差v
v=σ²
   * ±1σ=68.26%
   * ±2σ=95.46%
   * ±3σ=99.73%
   * 两个路径的标准差：(σ1²+σ2²)开根号

EAC = AC + ETC

EAC = AC + (BAC - EV)/x

x = 1 非典型  剩余工作按照 **计划绩效**
x = CPI 典型 剩余工作按照 **当前绩效** **默认**
x = CPI * SPI 同时考虑CPI和SPI的影响
x = 新绩效 制定绩效
x = SPI 计算时间的EAC


#### 完工尚需绩效指数⭐⭐⭐⭐
TCPI完工尚需绩效指数 = 剩余工作的预算价值/剩余预算

TCPI = (BAC - EV)/(BAC - AC)  BAC可行
TCPI = (BAC - EV)/(EAC - AC)  BAC不可行用EAC代替

TCPI > 1 很难完成
TCPI = 1 正好完成
TCPI < 1 很容易完成  值越小越好

完工偏差VAC = BAC - EAC
VAC > 0 没有超过预算
VAC < 0 超过预算

#### 一些财务概念
* 现值Present Value：未来的钱在现在的价值
* 折现率Discount Rate：将未来有限期预期收益折算成现值的比率
PV = FV(投资的终值)/(1 + r)n

* 净现值Net Present Value：时间段内报酬现值的总和减去原始投资额，**越大越好**
NPV > 0 接受该项目
NPV < 0 放弃该项目

多个备选方案的互斥决策中，净现值已经考虑了货币的时间价值，选择NPV正值中的最大者

* 内部收益率IRR(Internal Rate of return)  净现值等于零时的折现率，代表项目的抗风险（通货膨胀等）能力的大小，**越大越好**

* 静态投资回收期Payback Period:从项目投建之日起，用项目所得的净收益偿还原始投资额所需要的年限，**不考虑货币时间价值，越短越好**

* 投资回报率(Return On Investment):  年平均利润/投资总额 * 100%
不考虑货币时间价值，年平均利润是全部的利润。**越大越好**

* 效益成本比BCR(Bennfit-Cost Ratio)：项目投资与效益之间关系的比率 收益/投资
BCR > 1 接受该项目
BCR < 1 放弃该项目
**越大越好**

### 合同类别
* 项目范围划分
    * 总承包合同
        * 与同一个卖方签订
        * 不意味只签订一个总合同
    * 单项承包合同
        * 不同卖方，吸引更多卖方参与
    * 分包合同
        * 合同约定
        * 买方认可
        * 卖方将承包项目的一部分或几个部分（非项目的主体结构）再发包给**具有相应资质条件**的分包方，**不能转包整个项目**，**不能再次分包**
        * 如果分包的项目出现问题，买方既可要求**卖方**，也可直接要求**分包方**承担责任
* 项目付款方式划分
    * 总价合同Fixed-Price contract，已明确定义需求，不会出现重大范围变更
        * 固定总价合同Firm Fixed Price,**最常用**，**甲方最喜欢**
            * 价格确定，不允许改变（**除非工作范围发生变更**）
            * **卖方承担**因不良绩效导致的任何成本增加
            * 买方应该准确定义拟采购的产品和服务，对采购规范的任何变更都会增加买方的成本
        * 总价加经济价格调整合同FPEPA
            * 卖方履约期将**跨越几年**
            * 将以**不同货币**支付价款
            * 条件变化（**通货膨胀，某些特殊商品的成本增加**），以事先确定的方式对价格进行调整
        * 订购单（单边合同）
            * **非大量**采购**标准化产品**，**买方直接填写**卖方提供的订购单，**卖方照此供货**
        * 总价加激励费用合同FPIF Fixed Price Incentive Fee
            * 允许一定的绩效偏离
            * 对实现目标给与相关的财务奖励
            * 要设置**价格上限**，**卖方**必须完成工作并且要承担高于上线的全部成本
        * 计算
            * 总价 = 实际成本 + [目标利润 + (目标成本 - 实际成本)*卖方应承担比例]
            * 最终总价 = 总价>=最高限价？最高限价：总价
            * 实际利润 = 最终总价 - 实际成本

    * 成本补偿合同Cost-Reimbursable Contract:**全部合法实际成本**(可报销成本),**外加一笔费用**作为卖方的利润
        * 工作范围预计会发生重大变更
        * 成本加固定费用合同(CPFF) Cost Plus Fixed Fee，**乙方喜欢**
            * 买方报销卖方**一切可列支出成本**，并支付一笔**固定费用**（为初始成本估算的某一个比例）
        * 成本加奖励费用（CPAF） Cost Price Award Fee
            * 买方报销一切合法成本，**买方主观判断**来决定奖励费用，并且通常不允许申诉
        * 成本加百分比合同（CPPC），美国法律已经禁止使用
        * 成本加激励费用（CPIF） Cost Plus Incentive Fee
            * 买方报销卖方**一切可列支出成本**，卖方达到合同规定的绩效目标时，向卖方支付事先确定的**激励费用**
            * 最终成本不等于原始成本时，按照事先约定好的成本分摊比例来**分享节约**或**分担超出**部分
            * 不存在总价上的最高限价，有时候会有**利润的最高限价**和**最低限价**
        * 计算
            * 利润 = 目标利润 + （目标成本 - 实际成本）* 卖方应承担比例
            * 最终利润
                * 利润>=利润上限：最终利润 = 利润上限
                * 利润下限<利润<利润上线：最终利润 = 利润
                * 利润<=利润下限：最终利润 = 利润下限
            * 实际总价 = 最终利润 + 实际成本
    * 工料合同T&M/时间和手段合同
        * 兼具成本补偿合同和总价合同某些特点的**混合型**合同
        * 适用于**无法快速编织出准确的工作说明书，金额小，工期短，不复杂的项目**
        * 一般会在合同中规定**最高价值**和**时间限制**，防止成本无线增加
        * 预先设定单位人力或材料费率（包含卖方利润）
        * **买方承担工作量变动的风险，卖方承担单价风险**

### 合同的选择标准
